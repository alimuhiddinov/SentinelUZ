Getting Started

SentinelUZ is an endpoint monitoring system that watches what programs are running on your organisation's Windows computers. It detects suspicious activity — like known malware, connections to attacker-controlled servers, or misuse of built-in Windows tools — and alerts you so you can take action.

How Monitoring Works

1. A small program (the ‘agent’) runs on each Windows computer you want to monitor. It checks what processes are running and what network connections are active every 30 seconds.

2. The agent sends this information to your SentinelUZ server.

3. The server compares this data against a database of known threats — over 209,000 blacklisted IP addresses and 1,300+ known malware file signatures.

4. If a match is found, or if suspicious behaviour is detected, an alert appears on your dashboard.

Installation

What you need: A Windows 10 or later computer, the SentinelUZ agent file (edr_client.exe), and administrator access on the target computer.

1. Copy the agent folder to the target computer.

2. Double-click edr_client.exe.

3. Click ‘Yes’ on the Windows security prompt (UAC).

4. The agent installs itself as a Windows Service and starts monitoring automatically. It will start on every boot — no further action needed.

Alert Severity Guide

Every alert has a severity level that tells you how urgent it is. Here's what each level means and what you should do:

CRITICAL

Immediate threat detected
A known malware file or active attacker tool has been found on one of your computers. This requires immediate attention.
Examples: Known malware hash match, connection to a command-and-control server, known attack tool running.
Action: Isolate the affected computer from the network immediately. Do not shut it down — this preserves evidence.

HIGH

Likely malicious activity
A suspicious pattern has been detected that strongly suggests an attack is in progress or being prepared.
Examples: Connection to a blacklisted IP address, ransomware preparation tools running, suspicious process chains.
Action: Investigate within the hour. Check the process chain to understand what triggered the alert.

MEDIUM

Suspicious but not confirmed
A Windows tool commonly misused by attackers was launched from an unusual location or parent process.
Examples: PowerShell launched by Microsoft Word, command prompt opened by a PDF reader.
Action: Review during your next check-in. Verify whether the activity was expected.

LOW

Informational
A built-in Windows tool that can be misused was detected, but without additional suspicious context.
Examples: cmd.exe or PowerShell running on its own (no suspicious parent process).
Action: No immediate action needed. Review if it appears repeatedly on the same computer.

What is a LOLBin?

LOLBin stands for “Living Off the Land Binary”. These are legitimate Windows tools that are pre-installed on every Windows computer. Attackers misuse them because security software often trusts these programs — they're part of Windows itself.

SentinelUZ monitors 17 LOLBins. Here's what each one does normally, and how attackers misuse it:

Tool Normal Use How Attackers Misuse It
cmd.exeWindows command promptRuns malicious scripts silently in the background
powershell.exeWindows automation and scripting toolDownloads and runs malware from the internet without saving files to disk
pwsh.exePowerShell Core (cross-platform version)Same as powershell.exe — used for fileless attacks
wscript.exeRuns Windows Script files (.vbs, .js)Executes malicious scripts disguised as documents or updates
cscript.exeConsole-based Windows Script hostRuns malicious scripts from the command line
mshta.exeRuns HTML-based applicationsExecutes malicious code hidden inside HTML files — used by Bloody Wolf in Uzbekistan

Reading a Process Chain

When SentinelUZ detects a suspicious process, it shows you the “process chain” — a family tree of programs that explains how the suspicious program was started. Here is an example:

explorer.exe PID: 1204 ← Desktop shell (normal)
WINWORD.EXE PID: 3456 ← Microsoft Word (normal)
cmd.exe PID: 7891 LOLBIN ← Command prompt (suspicious!)
powershell.exe PID: 8012 SUSPICIOUS ← PowerShell (attack pattern!)

Why is this suspicious?

Microsoft Word should never open a command prompt. This pattern is common in phishing attacks — a malicious document tricks Word into running commands.

What SentinelUZ flags:

• cmd.exe gets a LOLBIN badge because it's a commonly misused Windows tool.

• The chain gets a SUSPICIOUS badge because PowerShell launched from a command prompt that was started by Word is a known attack pattern.

• The alert severity is HIGH because of the suspicious parent chain.

What To Do When You See An Alert

When an alert appears on your dashboard, follow these steps:

1
Don't panic
Alerts mean SentinelUZ noticed something worth investigating. Not every alert means you've been hacked — some are false positives from legitimate activity.
2
Check the severity
CRITICAL alerts need immediate attention. LOW alerts can wait for your next regular check-in. See the Severity Guide above.
3
Open the alert detail page
Click ‘Investigate’ on the alert card. This opens a detailed view with five tabs of information.
4
Look at the Process Chain tab
This shows you which program started the suspicious activity. If the chain looks like normal work (e.g., you were running a script yourself), it may be a false positive.
5
Check the Network tab
If the alert involves a network connection, check whether the IP address is known to your organisation. Internal IPs (192.168.x.x, 10.x.x.x) are usually safe.
6
Take action
• If it looks malicious: Disconnect the computer from the network. Contact your IT security provider or CERT.uz.
• If it's a false positive: Click ‘Acknowledge’ to mark the alert as reviewed. Consider adding an exclusion rule if it triggers alerts regularly.
• If you're not sure: Leave the alert unacknowledged and escalate to someone with more experience.

Managing Exclusion Rules

Exclusion rules tell SentinelUZ to stop alerting on a specific program. Use them for known-safe software that keeps triggering false positives.

Be careful: every exclusion rule is a blind spot. Only exclude programs you are certain are safe.

There are four match modes, from least safe to most safe:

NAME_ONLY LOW SAFETY
Matches any program with this filename. Warning: An attacker can name their malware ‘explorer.exe’ and it will be excluded. Only use this as a last resort.
NAME_AND_PATH MEDIUM
Matches the filename AND the folder it runs from. Better than name-only, but an attacker could place malware in the same folder.
HASH_ONLY HIGH
Matches the exact file signature (SHA256 hash). Even if an attacker uses the same filename, a different file won't match. Recommended for most exclusions.
ALL HIGHEST
All three must match: name, path, and hash. The strictest option. Use when you want to be absolutely sure.
Recommendation: Always use HASH_ONLY or ALL match mode. NAME_ONLY should be avoided — it's the easiest for attackers to bypass.

Threat Intelligence

SentinelUZ compares activity on your computers against a database of known threats. This database is called “threat intelligence” (TI). Here's what's loaded:

209,000+ Blacklisted IPs

IP addresses known to be used by attackers, malware command servers, and botnets.

Sources: IPsum, Feodo Tracker, ThreatFox

1,300+ Malware Hashes

File signatures (SHA256) of known malware families.

Sources: MalwareBazaar, targeted family imports

Targeted Families

Malware specifically relevant to Uzbekistan and Central Asia.

Families: NetSupport RAT, LockBit, BlackCat/ALPHV, Rhysida, Ajina.Banker

Regulatory Compliance

Uzbekistan's regulatory framework requires organisations to implement cybersecurity monitoring. SentinelUZ helps you meet these requirements:

PP-167 (Presidential Resolution, September 2023)

Mandatory cybersecurity measures for government and critical infrastructure organisations.
✓Endpoint monitoring — SentinelUZ provides real-time monitoring of all registered Windows endpoints.
✓Incident detection — Automated detection of known malware, suspicious process chains, and network connections to attacker infrastructure.
✓Audit trail — All alerts are stored indefinitely, providing a complete record of detected security events for auditors.
✓Threat intelligence — Active matching against 209,000+ known threat indicators.

PQ-153 (Cabinet Resolution, March 2024)

Technical requirements for information security systems in government networks.
✓Process monitoring — Every running process is recorded with its file hash, parent process, and command line.
✓Network monitoring — All TCP connections are tracked with source/destination IPs and ports.
✓Threat matching — Real-time comparison against government-recognised threat databases.
✓Data retention — Alert records kept indefinitely; raw telemetry retained per documented retention policy.
